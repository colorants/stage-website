Stage 23G

Competenties

Het belangrijkste deel - de competenties - hieronder vertel ik op welke manier ik welke competentie heb kunnen behalen.
Ik deel de competenties in twee stukken: stageplan en de daadwerkelijke uitkomst

  • 🛠️ Technische kennis en analyse :

    Ik upload mijn code in Github en ik maak gebruik van Pull Request, dit is de manier hoe ze het bij 23G doen. Zo kunnen andere developers meekijken met mijn code en tips geven, maar ook andersom. Ook heb ik mijn project gemaakt met de standaarden (boilerplate) die vanuit 23G zijn gemaakt. Hier hangt geen deadline aan vast, dit is de hele stage lang bezig. Implementeren, testen en opleveren. Aangezien de klant waarvoor ik ontwikkel op dezelfde werkvloer als ik ben kan ik snel testen en gemakkelijk dingen aanpassen. Bij bepaalde features of designkeuzes loop ik langs of stuur ik een berichtje, aan de hand van de feedback hiervan pas ik iets aan. De uiteindelijke oplevering willen ze hier daadwerkelijk gaan gebruiken en ik maak het dus met de aangeleverde standaarden.

    Deze competentie is mij volledig gehaald. Ik heb mijn project gemaakt met de standaarden van 23G en heb mijn code geüpload in Github. Hierdoor kunnen andere developers meekijken met mijn code en tips geven, maar ook andersom, ik kan ook meekijken met hun code en tips geven. Ook heb ik mijn project gemaakt met de standaarden die vanuit 23G zijn gemaakt. Aan het begin vond ik het moeilijk om nette code te schrijven maar naarmate de stage verliep ging dit steeds beter en wordt ik veel minder gecorrigeerd. Ik hoor ook van andere dat de pull request nakijken veel makkelijker gaat dan aan het begin.

  • 💡 Conceptualiseren :

    Door mij te verdiepen in bestaande platformen en de beste aspecten van deze te gebruiken en af te stellen voor wat de doelgroep (klant) graag wil, moet ik gebruikmaken van (voor mij) nieuwe technieken. Uit de opdracht komt uiteindelijk een MVP en deze wordt verder uitgewerkt.

  • 🖌️ Ontwerpen en prototypen :

    Na het opstellen van de user stories zijn er modules gemaakt met hoe en wat gemaakt moet worden, deze worden bijgehouden in Jira. De userstories maak ik stapsgewijs af waardoor het overzichtelijk blijft.

  • 🚀 Ondernemende houding en vaardigheden :

    Omdat ik bezig ben met een opdracht voor het bedrijf zelf en er momenteel nog geen vergelijkbaars bestaat, ben ik bezig met nieuwe oplossingen en probeer ik zelf nieuwe dingen uit. Zo gebruik ik de Github API om repositories te zien en link ik deze aan het aangeleverde klantenbestand en projecten. Als ik vind dat ‘feature x’ op een bepaalde manier geïmplementeerd moet gaan worden, werk ik hieraan zodat ik ook nieuwe stof leer maar het juist het product ook beter maar voor de uiteindelijke oplevering.

  • 🗣️ Projectmatig werken en communiceren :

    Aan mijn project werk ik individueel maar wel met bepaalde verwachtingen vanuit de stagebegeleider. Er zijn maandelijkse back-end meetings die ik bijwoon zodat ik ook weet wat er momenteel speelt en hoe deze verlopen. Ook kijk ik mee naar pull request op Github, hier kan ik meekijken naar de werkvorm van 23G. Als ik vragen heb rondom het project of een bepaalde pull request op Github, loop ik langs of stuur ik diegene een berichtje. Ik wil veel leren maar ben juist ook kritisch op waarom iemand iets op een bepaalde manier maakt.

  • 📚 Lerend vermogen :

    Door de pull requests en de wekelijkse feedback ronde ben ik goed bezig met mijn progressie en kijk ik ook hoe anderen feedback op elkaar geven. Ik leer een hele hoop en vind het ook leuk om hier veel te leren, doordat er nu nog geen grote verantwoordelijkheid aan hangt is het wel wat meer relaxt. Bij bepaalde nieuwe bevindingen sla ik deze goed op zodat ik bij een volgend project het op een andere manier doe.

  • 🌍 Maatschappelijke verantwoordelijkheid :

    Hier kan ik mij niet mee bezighouden omdat ik bij deze stage niet mee loop met het ui/ux gedeelte. Ik ben intern bezig en heb daarom weinig te maken met maatschappelijk of ethische implicaties. Wel ben ik aan het werk met veel gevoelige bedrijfsdata, bij 23G zijn ze ook ISO gecertificeerd. Door die certificatie houd ik mij ook bezig met het veiligstellen van deze data en ga ik er zorgvuldig mee om. Ik vind het belangrijk dat er met de data van bedrijven en klanten goed wordt omgegaan
