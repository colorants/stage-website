Project

Ik heb van begin tot eind een CRM (Customer Relationship Manager) gebouwd, waarin alle klanten en bijbehorende projecten centraal worden beheerd. Dit systeem is ontwikkeld om de salesafdeling van 23G te ondersteunen door hen sneller inzicht te geven in welke versies van een bepaald project worden gebruikt. Wanneer een repository meerdere versies achterloopt, biedt dit niet alleen verkoopkansen richting de klant, maar kan het ook de veiligheid van het systeem in gevaar brengen. Op dit moment worden de versies niet live bijgehouden, en het was mijn taak om dit proces te verbeteren. Met behulp van de GitHub API in combinatie met de EOL API worden alle repositories opgehaald, waarna de gebruikte versies automatisch worden uitgelezen. Het systeem is gebouwd met FilamentPHP, dat gebruik maakt van het Laravel Framework.

Dit project is een goede basis waarop nog veel extra features kunnen worden toegevoegd, zodat 23G klanten sneller en beter geholpen kan worden.
